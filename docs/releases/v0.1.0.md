# Release Management

## Version: v0.1.0

### Status
- 🚧 In Progress

### Objective
> Provide a foundational structure to model and manage plants and their containers, enabling creation, assignment, removal, and archiving of plants and containers.

### Summary
> This version sets up the core cultivation domain for the MVP.  
> - Allows modeling of plants and containers and their relationships  
> - Establishes CRUD operations for plants and containers  
> - Introduces domain events and read models for CQRS  
> - Provides a minimal GraphQL API for frontend integration  
> - Prepares multi-tenant support and soft delete for future expansion

---

## Scope (What’s Included)

### Backend
#### Domain
- Plant aggregate: creation, name, species, planted date, notes (optional)
- Plant status: `ACTIVE | ARCHIVED`
- Container aggregate: creation, type, name, location, metadata (optional)
- Relationship: assign/remove plant to/from container (1..N)

#### Application / Use Cases
- Create plant
- Create container
- Assign plant to container
- Remove plant from container
- Archive plant

#### Infrastructure
- TypeORM for write model (Postgres)
- MongoDB for read projections (CQRS)
- Multi-tenant support (DB or schema per tenant)
- Soft delete technical (not exposed in API)

#### Read Model / Projections
- `PlantView`: id, name, species, status, container (id + name), location
- `ContainerView`: id, name, type, location, plants[] (id + name)

#### API / Interfaces (GraphQL)
- **Mutations**
  - `createPlant(input: CreatePlantInput): Plant`
  - `archivePlant(id: ID!): Plant`
  - `createContainer(input: CreateContainerInput): Container`
  - `assignPlantToContainer(plantId: ID!, containerId: ID!): Plant`
  - `removePlantFromContainer(plantId: ID!, containerId: ID!): Plant`
- **Queries**
  - `plants: [Plant]`
  - `containers: [Container]`
  - `container(id: ID!): Container`

---

### Frontend
#### Views / UI
- List of plants
- List of containers
- Container detail showing assigned plants

#### Actions / Features
- Create plant
- Create container
- Assign / remove plant from container
- Archive plant

#### API Integration
- Connect to backend GraphQL API
- Handle queries and mutations for listed actions

## Explicitly Out of Scope (What’s NOT Included)

- Watering or care actions
- Sensors or automation
- Advanced agronomic rules
- History, notifications, or AI
- Photo management
- Calendars or schedules
- Import/export functionality
- Complex editing of entities

---

## Domain Events

This version emits or introduces the following domain events:

- `PlantCreated`
- `PlantArchived`
- `PlantAssignedToContainer`
- `PlantRemovedFromContainer`
- `ContainerCreated`

---

## Acceptance Criteria

This version is considered complete when:

- [ ] Backend aggregates, read models, and GraphQL API are functional
- [ ] Frontend views allow listing, creating, assigning, removing, and archiving plants/containers
- [ ] Multi-tenant support works for both backend and frontend
- [ ] Soft delete / archive is correctly handled in backend and reflected in frontend

---

## Technical Notes / Decisions

Relevant architectural or technical decisions taken for this version:

- Domain models decoupled from persistence (TypeORM + CQRS)
- Soft delete implemented technically, not exposed
- Multi-tenant architecture applied (DB or schema per tenant)
- GraphQL used as single API layer for frontend
- Minimal UI for MVP; focus on functionality
